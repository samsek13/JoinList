# JoinList - 时间均衡歌单混音器

这是一个帮你生成“完美均衡”歌单的小工具。比如你有 3 个喜欢的歌单，想把它们合成为一个新的，并且保证这 3 个歌单在新列表里的播放总时长完全一样（例如各占 20 分钟），那么这个工具就是为你准备的。

---

## 🚀 零基础保姆级启动指南 (Windows/Mac)

如果你不懂代码，也没关系。只要按照下面的步骤一步步操作，就能在自己的电脑上运行这个工具。

### 第一步：安装必备软件 (Node.js)

就像玩游戏需要安装环境包一样，运行这个工具需要安装 **Node.js**。

1.  访问 Node.js 官网：[https://nodejs.org/](https://nodejs.org/)
2.  下载左边的 **LTS 版本** (长期支持版)，一路点击“下一步”安装即可。
3.  **验证安装**：
    *   按下键盘 `Win + R`，输入 `cmd` 回车（Mac 用户打开终端 Terminal）。
    *   输入 `node -v` 并回车。
    *   如果屏幕上出现类似 `v20.x.x` 的版本号，说明安装成功！

### 第二步：准备项目文件

1.  **下载代码**：
    *   如果你是从 GitHub 克隆的，请直接进入项目文件夹。
    *   如果是下载的压缩包，请解压到一个你找得到的地方（比如 D盘的 Projects 文件夹）。
2.  **打开命令窗口**：
    *   **Windows 用户**：进入解压后的 `JoinList` 文件夹，在顶部的地址栏输入 `cmd` 并回车，会弹出一个黑色的窗口。
    *   **Mac 用户**：在文件夹上右键选择“在终端中打开”。

### 第三步：安装依赖与配置

在刚才打开的黑色窗口里，**依次**复制粘贴下面的命令并回车（请耐心等待每一步完成）：

1.  **安装依赖包** (自动下载运行所需的素材)：
    ```bash
    npm install
    ```
    *等待进度条跑完，如果不报错就成功了。*

2.  **配置环境文件** (生成默认设置)：
    *   **Windows 用户请执行**：
        ```bash
        copy .env.example .env
        ```
    *   **Mac/Linux 用户请执行**：
        ```bash
        cp .env.example .env
        ```
    *   *说明：这一步是创建配置文件，我们已经预设好了适合个人使用的配置，无需修改。*

3.  **初始化数据库** (创建存放数据的仓库)：
    ```bash
    npm run prisma:migrate
    ```
    *当你看到绿色的 "All migrations have been successfully applied" 字样时，表示数据库准备好了。*

### 第四步：启动服务！

输入最后一条命令来启动程序：

```bash
npm run dev
```

当看到屏幕上显示：
> Server running on http://localhost:3000

**恭喜你！🎉 你成功了！**

### 第五步：开始使用

1.  保持那个黑色窗口不要关闭（关了服务就停了）。
2.  打开浏览器（推荐 Chrome 或 Edge）。
3.  在地址栏输入：[http://localhost:3000](http://localhost:3000)
4.  现在你可以开始使用歌单混音工具了！

## 常见问题 (FAQ)

**Q: 启动时提示端口被占用 (EADDRINUSE)？**
A: 这说明你的电脑上已经有程序占用了 3000 端口。
1. 用记事本打开项目里的 `.env` 文件。
2. 找到 `PORT=3000`，把它改成 `PORT=3001` 或其他数字。
3. 保存文件，重新运行 `npm run dev`。
4. 浏览器访问地址也要相应改成 `http://localhost:3001`。

**Q: 怎么停止服务？**
A: 在黑色窗口里按键盘上的 `Ctrl + C`，然后输入 `Y` 确认即可。

**Q: 我需要安装 Redis 吗？**
A: **不需要**。默认配置使用的是“内存模式 (Memory Mode)”，非常适合个人本地使用。只有在服务器上大规模部署时才需要 Redis。
