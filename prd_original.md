产品需求文档 (PRD)：时间均衡歌单混音器 (V1.0)
=============================

1. 产品概述

-------

### 1.1 产品定义

一款 Web 应用工具，允许用户输入多个网易云音乐歌单，根据用户设定的总时长，从各个源歌单中**以时长为基准**等比例、随机地抽取歌曲，生成一个新的均衡歌单。

### 1.2 核心价值

解决“歌单偏好失衡”问题。当用户想混合多个不同风格或来源的歌单时，传统的随机播放往往会导致数量多的歌单占据主导，而本应用确保每个源歌单在**收听时间**上是绝对公平的。

* * *

2. 用户流程 (User Flow)

-------------------

1. **登录验证**：用户通过扫码或 Cookie 形式登录网易云音乐账号（获取创建歌单权限）。

2. **输入配置**：
   
   * 粘贴 2-10 个源歌单 (OldLists) 链接。
   
   * 设定目标新歌单的最大总时长 $T_{max}$。

3. **解析与计算**：系统抓取歌曲元数据，执行去重与均衡算法。

4. **预览与确认**：展示即将生成的歌单列表、各来源时长占比、由于供应不足导致的实际总时长缩减说明。

5. **执行创建**：在用户账号下自动新建歌单并导入歌曲。

* * *

3. 核心功能需求

---------

### 3.1 数据获取与预处理

* **支持上限**：最多支持 10 个源歌单，每个歌单最多 1000 首歌。

* **元数据抓取**：需获取每首歌的 `ID`、`标题`、`歌手`、`时长 (Duration)`。

* **去重逻辑**：
  
  * **判定标准**：`标题` + `第一歌手` 完全一致即视为重复。
  
  * **归属权**：采取“先到先得”原则。若歌单 A 和 B 均有歌曲 X，则 X 仅保留在 A 的可选池中，B 的可选池剔除该歌曲。

### 3.2 均衡抽取算法 (Core Algorithm)

算法需满足“绝对时长平等”约束：

1. **计算可用总量**：统计每个源歌单去重后的总时长 $D_i$。

2. **确定单源目标时长** ($t_{target}$):
   
   * 初始目标：$t_{initial} = T_{max} / N$ （$N$ 为歌单数量）。
   
   * 最终目标：$t_{target} = \min(t_{initial}, D_1, D_2, \dots, D_n)$。

3. **抽取逻辑**：
   
   * 对每个源歌单，将其歌曲池随机洗牌 (Shuffle)。
   
   * 按洗牌后的顺序累加歌曲时长，直到总和最接近且不超过 $t_{target}$。
   
   * **容错**：允许最终总时长与目标时长存在少量偏差（如单首歌曲时长的量级）。

### 3.3 交互反馈

* **异常提示**：若某个源歌单因去重后变为空值或极短，需实时提醒用户。

* **时长缩减通知**：若因某个源歌单太短导致总时长未达到 $T_{max}$，在生成前必须明确提示：“由于歌单 [X] 长度限制，新歌单总时长调整为 [Y] 分钟”。

* * *

4. 技术规格

-------

### 4.1 技术栈建议

* **前端**：Next.js 或 Vue3 (提供简洁的单页 UI)。

* **后端**：Node.js (由于网易云 API 社区生态以 Node.js 最为成熟)。

* **接口库**：使用 `NeteaseCloudMusicApi` (Binaryify 版) 进行模拟登录和歌单操作。

### 4.2 数据结构 (JSON 示例)

JSON
    {
      "config": {
        "max_total_duration": 10800, // 3小时，单位秒
        "source_lists": ["url1", "url2"]
      },
      "results": {
        "actual_total_duration": 9000,
        "distribution": [
          {"source": "List A", "contributed_time": 4500, "song_count": 15},
          {"source": "List B", "contributed_time": 4500, "song_count": 12}
        ]
      }
    }

* * *

5. 风险与限制

--------

* **接口稳定性**：非官方 API 可能因网易云风控策略变动而失效，需具备更新 Cookie 的功能。

* **频率限制**：短时间内抓取大量歌单（10个 x 1000首）可能触发网易云的频率限制，建议在后端做请求节流（Throttling）。

* **版权限制**：部分歌曲可能在用户账号下无版权，无法添加至新歌单。

* * *

6. 后续迭代 (Roadmap)

-----------------

* **V1.1**：支持 Spotify 单平台均衡。

* **V2.0**：支持跨平台匹配（Spotify 导入网易云）。

* **V2.1**：增加“权重分配”，允许用户设置 A 歌单占 20% 时长，B 歌单占 80% 时长。
