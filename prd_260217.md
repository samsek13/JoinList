# 产品需求文档 (PRD)：JoinList - 时间均衡歌单混音器 (V1.0 - 20260217版)

## 1. 产品概述

### 1.1 产品定义
JoinList 是一款 Web 应用工具，旨在解决“多歌单混合时的时长分配不均”问题。它允许用户输入多个网易云音乐歌单链接，设定一个目标总时长，系统将从每个源歌单中**以时长为基准**等比例、随机地抽取歌曲，生成一个新的、时间分布均衡的网易云音乐歌单。

### 1.2 核心价值
*   **绝对公平**：确保每个源歌单在最终成品中的播放时长占比一致（例如：3个源歌单，各占 1/3 时长）。
*   **随机惊喜**：在满足时长约束的前提下，随机打乱抽取歌曲，每次生成的结果都不同。
*   **自动化**：一键完成抓取、去重、计算、创建歌单的全流程。

---

## 2. 用户流程 (User Flow)

1.  **访问与登录**：
    *   用户访问首页。
    *   提供网易云音乐的 `MUSIC_U` Cookie（或完整 Cookie 字符串）进行身份验证。
    *   *注：目前版本采用直接输入 Cookie 的方式，后续可扩展二维码登录。*

2.  **配置任务**：
    *   **源歌单**：输入 2-10 个网易云音乐歌单链接（支持网页链接或 ID）。
    *   **目标时长**：设定新歌单的最大总时长（例如 60 分钟）。

3.  **提交处理**：
    *   点击“开始生成”。
    *   系统后台异步处理：校验链接 -> 抓取歌单详情 -> 歌曲去重 -> 均衡算法计算 -> 创建新歌单。

4.  **进度反馈**：
    *   前端轮询任务状态，实时展示进度条（0% -> 100%）。
    *   展示当前正在处理的步骤（如“正在分析歌单 A...”、“正在创建...”）。

5.  **结果展示**：
    *   任务完成后，显示生成的歌单链接（点击跳转网易云）。
    *   展示**时长分布报告**：列出每个源歌单实际贡献的时长和歌曲数量。
    *   若有错误（如 Cookie 失效、歌单为空），展示具体错误信息。

---

## 3. 核心功能需求

### 3.1 数据获取与预处理
*   **链接解析**：智能识别多种格式的网易云歌单链接（URL 参数 id, 纯数字 ID）。
*   **元数据抓取**：调用 Netease API 获取歌曲 ID、标题、歌手、时长。
*   **去重逻辑**：
    *   **指纹算法**：`normalizeSign = 标题(小写) + "##" + 第一歌手(小写)`。
    *   **去重策略**：全局去重。如果歌曲 X 出现在多个源歌单中，仅在第一个遇到的歌单中保留，后续歌单中剔除，防止重复计算时长。

### 3.2 均衡抽取算法 (Mixer Engine)
为了实现“时长平等”，算法逻辑如下：

1.  **计算单源配额 ($t_{target}$)**：
    *   $t_{base} = \text{用户设定总时长} / \text{源歌单数量}$
    *   $t_{limit} = \min(\text{各源歌单实际总时长})$
    *   $t_{target} = \min(t_{base}, t_{limit})$
    *   *解释：每个源歌单的目标时长不能超过平均值，也不能超过任何一个源歌单本身的总长度（木桶效应）。*

2.  **随机抽取**：
    *   对每个源歌单的可用歌曲池进行随机洗牌 (Shuffle)。
    *   按顺序累加歌曲时长，直到即将超过 $t_{target}$ 为止。

3.  **结果聚合**：
    *   收集所有被选中的歌曲 ID。
    *   统计实际的总时长和各部分占比。

### 3.3 任务队列系统
*   **异步设计**：由于歌单抓取和创建耗时较长（可能几十秒），必须采用异步任务队列。
*   **双模式支持**：
    *   **Redis 模式**：生产环境使用，持久化、高可靠。
    *   **Memory 模式**：开发环境或轻量部署使用，无外部依赖。

---

## 4. 数据模型

### 4.1 User (用户)
*   `id`: 唯一标识（Cookie 的哈希值）。
*   `cookie`: 网易云 Cookie（加密或原始存储）。
*   `platform`: 平台类型（目前固定为 "netease"）。

### 4.2 Task (任务)
*   `id`: UUID。
*   `ownerId`: 关联用户。
*   `status`: 状态机 (Pending -> Processing -> Completed/Failed/NeedAuth)。
*   `progress`: 进度百分比 (0-100)。
*   `configJson`: 存储输入参数（源歌单、目标时长）。
*   `resultUrl`: 最终生成的歌单链接。
*   `distributionJson`: 结果统计数据。

---

## 5. 限制与边界情况
*   **Cookie 有效性**：若 Cookie 过期或 IP 限制，API 会返回 301 或错误，系统需标记任务为 `NeedAuth` 或 `Failed`。
*   **歌单隐私**：只能读取公开歌单或用户自己创建的私密歌单。
*   **API 频率**：网易云 API 有频率限制，抓取大歌单时需分批处理并添加随机延迟 (Sleep)。
*   **空结果**：若去重后某歌单无歌可用，任务应报错提示。
